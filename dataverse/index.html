1 What is Microsoft Dataverse?

Microsoft Dataverse lets you securely store and manage data used by business applications. Data is organized into tables (rows and columns), and Dynamics 365 apps like Sales and Customer Service are built directly on top of it. It comes with a rich standard schema — the Common Data Model — that you extend with your own tables.

Why it mattersBecause every CRM app shares one Dataverse, a customer record created in Sales is the same record a Service agent sees — no syncing, no duplication.

2 Core concepts

TermMeaning
Table (entity)A set of rows storing one kind of record (Account, Contact, Case).
Column (field)A single attribute of a row (name, email, status).
Row (record)One instance — e.g. the account "Contoso".
RelationshipA link between tables (an Account has many Contacts).
Choice (option set)A field with a fixed set of values.
SolutionA package of customizations you move between environments.

3 Model-driven apps

A model-driven app builds its UI from the data model. You define tables, forms, views, and relationships; the app generates a consistent, responsive interface automatically — no pixel-by-pixel design.

Forms

How a single record is viewed and edited.

Views

Filtered, sortable lists of rows.

Site map

The app's navigation across tables and pages.
Component-focused, not canvasModel-driven apps are ideal for complex, data-dense business processes. Choose canvas apps only when you need pixel-perfect, task-specific UI.

4 Security model

Dataverse security is layered, so users see exactly what their role allows:

LayerControls
EnvironmentWho can access the whole environment.
Security roleWhat a user can do (create/read/write/delete) per table.
Business unitOrganizational scoping of data ownership.
Row & field levelSharing individual rows; hiding specific columns.
Least privilegeGrant the narrowest role that lets a user do their job. Broad roles leak data across business units fast.
